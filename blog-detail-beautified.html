Node项目使用log4js进行日志记录

2022-09-11 06:57:00 Node.js node, koa 1,248 阅读 8 评论

在Node.js项目中，日志记录是一个重要的环节，它可以帮助我们追踪应用程序的运行状态、排查问题以及监控系统性能。本文将详细介绍如何在Node项目中使用log4js进行高效的日志管理。

安装与配置

目前log4js的最新版本是6.6.1，我们可以通过npm进行安装：

npm install log4js --save

log4js的日志默认从info级别开始，日志内容由时间、级别和具体信息组成。log4js可配置的级别一共有九个，下面是一个级别参考图：

基本使用

在项目中引入log4js并进行基本配置： 

const log4js = require('log4js');

log4js.configure({
  appenders: {
    console: { type: 'console' },
    file: { type: 'file', filename: 'app.log' }
  },
  categories: {
    default: { appenders: ['console', 'file'], level: 'info' }
  }
});

const logger = log4js.getLogger();

// 使用不同级别的日志
logger.trace('Entering cheese testing');
logger.debug('Got cheese.');
logger.info('Cheese is Gouda.');
logger.warn('Cheese is quite smelly.');
logger.error('Cheese is too ripe!');
logger.fatal('Cheese was breeding ground for listeria.');

高级配置

对于生产环境，我们通常需要更复杂的配置，例如按日期分割日志文件： 

log4js.configure({
  appenders: {
    console: { type: 'console' },
    dateFile: {
      type: 'dateFile',
      filename: 'logs/app.log',
      pattern: '-yyyy-MM-dd',
      compress: true
    }
  },
  categories: {
    default: { appenders: ['console', 'dateFile'], level: 'info' },
    error: { appenders: ['dateFile'], level: 'error' }
  }
});

结合Koa框架使用

在Koa应用中集成log4js中间件： 

const Koa = require('koa');
const app = new Koa();
const log4js = require('log4js');

// 配置log4js
log4js.configure({
  appenders: {
    access: { type: 'dateFile', filename: 'logs/access.log' },
    app: { type: 'file', filename: 'logs/app.log' },
    errorFile: { type: 'file', filename: 'logs/errors.log' }
  },
  categories: {
    default: { appenders: ['app'], level: 'info' },
    access: { appenders: ['access'], level: 'info' },
    error: { appenders: ['errorFile'], level: 'error' }
  }
});

// 创建日志中间件
const logger = log4js.getLogger('app');
const accessLogger = log4js.getLogger('access');

// 记录HTTP请求
app.use(log4js.connectLogger(accessLogger, { level: 'auto' }));

// 应用中间件
app.use(async (ctx, next) => {
  try {
    await next();
  } catch (err) {
    logger.error(err);
    ctx.status = err.statusCode || 500;
    ctx.body = { error: err.message };
  }
});

app.listen(3000);

常见问题与解决方案

在使用log4js过程中，我们可能会遇到一些问题，下面是一些常见问题及解决方案：

  1. 日志文件过大：可以配置文件大小分割或日期分割策略
  2. 多进程应用日志混乱：使用cluster模式时需要特殊配置
  3. 性能问题：避免在高并发场景下使用同步写入方式
日志记录是系统监控和问题排查的重要工具，合理使用log4js可以帮助我们更好地管理应用程序的运行状态。
